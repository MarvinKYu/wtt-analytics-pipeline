### Work Completed

#### feat: dbt staging/intermediate/mart models (§7.9–7.13)

##### stg_matches.sql
- Deduplicates bronze_wtt_matches by (player_id, match_id) keeping latest scraped_at
- Casts all columns to correct types (INT64, TIMESTAMP)
- Classifies event_type: WTT | ITTF | OTHER based on event_name prefix/keywords
- Filters out rows with null match_id or match_year

##### stg_rankings.sql
- Deduplicates bronze_ittf_rankings by (player_id, snapshot_date) keeping latest scraped_at
- Parses snapshot_date STRING → DATE via SAFE.PARSE_DATE
- Filters out rows with null player_id or snapshot_date

##### stg_ranking_history.sql
- Deduplicates bronze_ittf_ranking_history by (player_id, week_date) keeping latest scraped_at
- Parses week_date STRING → DATE via SAFE.PARSE_DATE
- Handles pre-2014 records with approximated month-start dates

##### int_player_match_history.sql
- Resolves each stg_matches row (queried player_id + match) to player_id, opponent_id, result
- Name-to-player_id lookup: latest name per player_id from stg_rankings → lowercased lookup dict
- Position detection: compares lower(player name) with lower(player_a_name) / lower(player_x_name)
- Result computed from game counts per position (A: result_a_games > result_x_games → WIN)
- Rows where position cannot be determined (name formatting mismatch) are excluded via WHERE result IS NOT NULL
- opponent_id is NULL when the opponent is not in the rankings seed list (valid for replay.py)

##### mart_player_ratings.sql
- Joins mart_player_ratings_raw with latest stg_rankings snapshot (INNER JOIN, only ranked players)
- Computes implied_rank = row_number() over (partition by gender order by rating desc)
- Computes divergence = ittf_rank - implied_rank (positive = ITTF overranks vs model)

##### mart_rating_vs_ranking.sql
- Filters mart_player_ratings to rd < 200 (high-confidence ratings only)
- Adds divergence_category: overranked (>+10), underranked (<-10), aligned (±10)
- Adds rating_percentile = percent_rank() within gender × 100, rounded to 1 decimal

##### schema.yml updates (all three layers)
- Added column descriptions and tests to all three schema.yml files
- Tests: not_null, unique, accepted_values for event_type, gender, result, divergence_category
